# Visual Lesson Builder Skill

Use this skill when creating or improving a learning module in the Claude Code AI Visual Lab app.

## Required output for every lesson module

Each module must include:

1. **One-liner**
   - One sentence.
   - Beginner-friendly.
   - No jargon unless explained visually.

2. **Metaphor**
   - Use a physical or visual metaphor.
   - Keep it consistent with the app's command-center/workshop language.

3. **Visual model**
   - Diagram, flow, gauge, map, or animated scene.

4. **Interaction**
   - Slider, toggle, drag/drop, click-to-connect, quiz, simulation, or prompt comparison.

5. **Concrete Claude Code example**
   - Show a real-ish workflow or command.
   - Keep it compact.

6. **Takeaway**
   - One short card: “Remember this.”

## Avoid

- Long documentation text.
- Generic cards with icons only.
- Too many concepts in one module.
- Interactions that are decorative but do not teach.
- Inaccurate simplification.

## Quality checklist

Before finishing a module, verify:
- A beginner can understand the core concept in under 60 seconds.
- The interaction changes something visible.
- The module is visually distinct but consistent.
- The copy is concise.
- The concept is technically accurate.

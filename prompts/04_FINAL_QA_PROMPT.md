# Final QA Prompt

Before calling the app finished, perform a full quality review.

Check:
- app runs locally
- no build/type errors
- no broken navigation
- every module has interaction
- text is concise
- visual metaphors are consistent
- layout is responsive
- animations are helpful, not distracting
- accessibility basics are covered
- concepts are accurate
- installed toolkit map is represented clearly
- final README instructions are accurate

Use:
- run / verify
- code-review
- design:accessibility-review
- design:design-critique
- frontend-design if visual quality is weak

Return:
1. pass/fail checklist
2. fixes applied
3. known limitations
4. next recommended improvements

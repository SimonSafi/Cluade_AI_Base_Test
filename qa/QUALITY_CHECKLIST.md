# Quality Checklist

## Product
- [ ] Home screen feels impressive.
- [ ] App does not feel like a documentation site.
- [ ] Visual metaphor is consistent.
- [ ] Navigation is clear.
- [ ] Modules are discoverable.

## Learning
- [ ] Claude Code is explained clearly.
- [ ] Agents/subagents are explained clearly.
- [ ] Skills are explained clearly.
- [ ] MCP is explained clearly.
- [ ] Tokens/context are explained clearly.
- [ ] Workflow orchestration is explained clearly.
- [ ] Each module has a short takeaway.

## Interaction
- [ ] Every module has an interaction.
- [ ] Interactions teach, not just decorate.
- [ ] Animations are smooth.
- [ ] No broken controls.

## Technical
- [ ] App runs locally.
- [ ] Typecheck passes.
- [ ] Build passes.
- [ ] No console errors.
- [ ] Components are reasonably organized.
- [ ] No giant unmaintainable files.

## Accessibility
- [ ] Text contrast is acceptable.
- [ ] Keyboard navigation basics work.
- [ ] Motion is not excessive.
- [ ] Buttons have clear labels.
- [ ] Layout is readable on laptop screens.

## Final report
- [ ] What was built.
- [ ] How to run it.
- [ ] Main files.
- [ ] Known limitations.
- [ ] Suggested next iterations.
